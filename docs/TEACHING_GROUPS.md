# مجموعات التدريس — دليل تشغيل

## المفاهيم

| المصطلح | المعنى |
|---------|--------|
| **حصة جدول** | صف في `schedule` (يوم + وقت) |
| **مجموعة تدريس** | طلاب + أستاذ + مقرر + قسم + فصل (A/B أو «الكل») |
| **single** | محاضرات متعددة لنفس الطلاب |
| **split** | مجموعات A/B منفصلة — يُختار عند التسجيل |

## متى تستخدم single vs split

- **single**: المقرر له عدة محاضرات في الأسبوع و**نفس الطلاب** يحضرون كلها (ميكانيكا: أحد + أربعاء).
- **split**: الطلاب **مقسّمون** على مجموعات حضور (A صباحاً، B مساءً).

## خطوات الإعداد (فصل جديد)

1. افتح **إعداد مجموعات التدريس** من صفحة الجدول.
2. **ترحيل حصص (single)** — ينشئ مجموعات افتراضية ويربط الحصص.
3. لكل مقرر **split**: اختر «مجموعات منفصلة» وعيّن A/B والأستاذ والحصص.
4. **ترحيل تسجيلات → مجموعات** — يربط التسجيلات الحالية (single تلقائياً).
5. **ترحيل تقييمات → مجموعات** — يربط تقييمات المقرر القديمة.
6. راجع تقرير التدقيق من نفس الصفحة.

## سطر أوامر (اختياري)

```bash
python scripts/backfill_teaching_groups.py
python scripts/backfill_teaching_groups.py --semester "خريف 44-45" --audit-only
python scripts/backfill_teaching_groups.py --skip-schedule --skip-registrations
```

## التقييم والاستبيانات

- الطالب يُقيّم **مرة واحدة لكل مجموعة تدريس** (وليس لكل محاضرة).
- عدد المسجّلين في التقارير = `COUNT` من `registrations.teaching_group_id`.
- تقرير «مجموعات بلا تقييم» يعتمد على `teaching_groups` وليس صفوف الجدول.

## الأستاذ والدرجات

- **مقرراتي** تعرض مجموعات تدريس (قسم + A/B + محاضرات).
- قائمة الدرجات تُحمّل الطلاب من مجموعة التدريس عند توفر `teaching_group_id`.

## استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| عدد طلاب خاطئ في التقرير | تأكد من ربط التسجيلات بمجموعة A/B |
| مقرر يظهر مرتين للأستاذ | راجع إعداد split — كل مجموعة بطاقة منفصلة |
| تقييم مكرر | تأكد من `teaching_group_id` في `course_evaluations` |
| حصص بلا مجموعة | شغّل ترحيل حصص أو اربط يدوياً من الإعداد |
