## 3) الرؤية النهائية — بعد اكتمال الخطة

### مثلث التقييم + شركاء

```
              الطالب (داخلي)
                 /    \
                /      \
         الخريج -------- القطاع (استراتيجي + تشغيلي)
                            |
                      الشريك المجتمعي
```

### الاستبيانات الخارجية المطلوبة (4)

| الرمز | الجمهور | الغرض | التكرار |
|-------|---------|--------|---------|
| `employer_strategic` | جهة عمل / قطاع | رأي في الرؤية والرسالة والخطة والمخرجات | كل 2–3 سنوات |
| `alumni` | خريج | رضا عن التعليم والجاهزية المهنية | سنة بعد التخرج |
| `employer_graduate` | جهة عمل (مشرف خريج) | جاهزية الخريج في العمل | سنوي (اختياري — مرحلة لاحقة) |
| `community_partner` | شريك تدريب / مجتمع | جودة الشراكة | سنوي |

### مبدأ التبسيط

- **داخلي:** 10 بنود Likert (كما هو).
- **خارجي:** **6 بنود Likert + سؤال مفتوح واحد** لكل استبيان.
- **حد أدنى للتجميع:** 5 مستجيبين (قابل للتعديل في القالب).

## 4) استشارة القطاع — بدون مجلس استشاري

> الكلية **لا تملك مجلساً استشارياً دائماً** — وهذا مقبول في ممارسات الجودة والاعتماد. المطلوب **استشارة منتظمة لأصحاب المصلحة**، لا هيكل باسم «مجلس».

### الآلية المعتمدة: «دورة استشارة القطاع»

```
قائمة جهات (15–25)  →  رابط دعوة  →  employer_strategic  →  تقرير  →  مدخل لمراجعة الخطة
```

| نوع الجهة | أمثلة | العدد |
|-----------|--------|-------|
| توظيف / موارد بشرية | شركات توظّف خريجين | 8–12 |
| تدريب تعاوني | مواقع التدريب | 5–8 |
| قطاع حكومي / منظمات | جهات ذات صلة بالتخصص | 3–5 |

**المصادر:** التدريب التعاوني، شؤون الخريجين، علاقات القطاع، مشاريع الأساتذة مع الشركات.

### مسؤول واحد (مالك العملية)

- عميد مساعد للجودة / منسق تدريب / رئيس قسم — **ليس لجنة**.
- المهام: تحديث القائمة، إرسال الروابط، متابعة الردود، تقديم ملخص للإدارة.

### صياغة للاعتماد

> «تجري الكلية **استشارة دورية لأصحاب المصلحة من القطاع** (جهات التوظيف وشركاء التدريب) عبر استبيان منظم و/أو لقاء مركز، لمراجعة ملاءمة الرؤية والأهداف الاستراتيجية ومخرجات التعلم. **لم يُنشأ مجلس استشاري دائم**؛ وتُخطَّط دورات الاستشارة كل [2–3] سنوات ضمن دورة ضمان الجودة.»

### قنوات اختيارية (حسب الجهد)

| القناة | الجهد | الاستخدام |
|--------|-------|-----------|
| رابط بريد فقط | منخفض | كافٍ للبداية والاعتماد |
| لقاء افتراضي 90 دقيقة | متوسط | قبل مراجعة الخطة الاستراتيجية |
| لجنة مراجعة مناهج مؤقتة | أعلى | عند مراجعة PLO أو اعتماد برنامج |

## 5) بنود الاستبيانات الخارجية (مبسّطة)

### `employer_strategic` — استشارة القطاع

#### الخطوة 1 — بطاقة تعريف جهة العمل
(انظر القسم أعلاه: نوع الجهة، الاسم، القطاع، المنصب، توظيف خريجين)

#### الخطوة 2 — لوحة هوية الكلية (قراءة فقط) ثم الأسئلة

**مصدر البيانات:** `college_identity` + `college_strategic_goals` + `college_graduate_outcomes` عبر  
`college_profile_payload()` — **بدون رموز أو اختصارات في واجهة المستجيب.**

| قسم اللوحة | المحتوى المعروض | قواعد العرض |
|------------|-----------------|-------------|
| **الرؤية** | نص `vision_ar` | كامل |
| **الرسالة** | نص `mission_ar` | كامل |
| **القيم الأساسية** | من `values_json` | عناوين + وصف مختصر (اختياري: قابل للطي) |
| **ملخص الخطة الاستراتيجية** | `strategic_plan_summary_ar` أو `intro_ar` | فقرة أو فقرتان — يُفضّل حقل مخصّص في هوية الكلية |
| **الأهداف الاستراتيجية للكلية** | الأهداف الرئيسية الثمانية فقط (`parent_code` فارغ) | **عناوين عربية فقط** — بلا رموز مثل IG1 |
| **تفاصيل الأهداف** | الأهداف الفرعية | داخل **أقسام قابلة للطي** — اختياري للمستجيب |
| **مخرجات تعلم الخريج على مستوى الكلية** | من `college_graduate_outcomes` | **عناوين عربية فقط** — بلا رموز مثل GLO1 |

**نص تمهيدي للمستجيب (مثال):**  
«اطلعوا أدناه على رؤية الكلية ورسالتها وأهدافها الاستراتيجية وملخص خطتها ومخرجات تعلم الخريج، ثم أجيبوا على الأسئلة من منظور قطاعكم.»

#### أسئلة الاستبيان (6 + مفتوح) — صياغة بلا اختصارات

| # | البند |
|---|--------|
| 1 | وضوح **رسالة الكلية** وهويتها المهنية لدى قطاعكم |
| 2 | ملاءمة **رؤية الكلية** لاحتياجات سوق العمل والتنمية |
| 3 | ملاءمة **الأهداف الاستراتيجية للكلية** لأولويات قطاعكم |
| 4 | ملاءمة **مخرجات تعلم الخريج على مستوى الكلية** لمتطلبات العمل الفعلية |
| 5 | وضوح **أولويات الخطة الاستراتيجية للكلية** من منظوركم الخارجي |
| 6 | استعدادكم للشراكة مع الكلية (تدريب تعاوني، مشاريع مشتركة، لقاء أو ورشة) |
| م | **مفتوح:** أهم ثلاث توصيات لتحسين الخطة الاستراتيجية أو مخرجات التعلم |

**ربط اعتماد (داخلي فقط — لا يظهر للمستجيب):** GV-01-1، QA-03-1، CR.

## 8) قرارات تصميم (نهائية)

| القرار | الاختيار |
|--------|----------|
| بنود الاستبيان الخارجي | 6 Likert + 1 مفتوح |
| مجلس استشاري | **لا** — «دورة استشارة قطاع» + قائمة جهات |
| حساب دائم للخارجي | **لا** — token فقط |
| فصل استراتيجي / تشغيلي لجهة العمل | **نعم** — استبيانان |
| ربط أهداف تفصيلي (40 هدف) | **لا في الأسئلة** — عرض في اللوحة فقط |
| اختصارات في واجهة المستجيب | **ممنوعة** — عربية كاملة (انظر قسم المصطلحات) |
| لوحة هوية في استشارة القطاع | **نعم** — رؤية + رسالة + أهداف + ملخص خطة + مخرجات خريج |
| خريج 5 سنوات كقالب منفصل | **لا** — فلتر سنة التخرج في التقارير |
| بيانات الخريج | **يُدخلها الخريج** (سنة، قسم، شعبة، عمل) |
| أنواع جهات العمل | **قالب واحد** — تصنيف في بطاقة التعريف (حكومي/خاص/عام/…) |
| إخفاء النتائج | حد أدنى 5 (قابل للتعديل) |
| دورة vs فصل | `semester` للداخلي؛ `cycle_label` للخارجي |

## 12) سياسة المصطلحات — واجهة المستجيب الخارجي

> **القاعدة:** أي نص يراه الخريج أو جهة العمل **لا يحتوي رموزاً أو اختصارات إنجليزية**.  
> الرموز تبقى **داخل النظام والتقارير الفنية** لربط الاعتماد فقط.

| لا تُستخدم في الاستبيان | استخدم بدلاً منها |
|-------------------------|-------------------|
| IG / IG1 | الأهداف الاستراتيجية للكلية |
| GLO / GLO1 | مخرجات تعلم الخريج على مستوى الكلية |
| PLO | مخرجات تعلم البرنامج الأكاديمي |
| CLO | مخرجات تعلم المقرر الدراسي |
| CR | الشراكات المجتمعية والتعاون مع القطاع |
| KPI | مؤشرات قياس الأداء |
| QA / GV / SS / FF / HR | *(لا تظهر للمستجيب)* — معايير الاعتماد داخلية |

**تنظيف البذرة:** مراجعة `college_identity_seed` وعناوين الأهداف لإزالة `(CLO/PLO)` و`(I/R/M)` من النصوص المعروضة للقطاع — أو تمريرها عبر طبقة عرض تُحذف الأقواس.
